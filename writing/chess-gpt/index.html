Chess-GPT: Human-Like Chess with a Fine-Tuned Language Model

Supervised bachelor thesis project at KTH

Chess-GPT grew out of bachelor thesis work by Edwin Ostlund and Aron Malmborg and asked a simple question: can a GPT-style model generate chess that feels recognizably human rather than merely legal?

Open the live demo | Run it on Hugging Face

Short version. Chess is a clean sequential decision-making problem, and this project explored whether autoregressive models trained on move sequences can capture enough structure, style, and imperfection to produce play that feels human.

Why this project is interesting

Chess is a natural sequential decision-making problem, but it is also a clean setting for asking whether autoregressive models can learn useful structure from move sequences alone. That makes it a good test case for bridging language modeling and strategic behavior.

What makes Chess-GPT more interesting than a standard chess engine is the goal. The question is not only whether the model can produce legal or strong moves, but whether it can produce play that feels recognizably human in style, pacing, and mistakes.

What the project explored

  • Representing chess games as token sequences suitable for GPT-style modeling.
  • Fine-tuning a language model on chess move data.
  • Evaluating generated play with an emphasis on human-like behavior, not only raw engine strength.
  • Turning the result into an interactive demo that can be tested directly in the browser.

Why it fits my broader interests

This project is more applied than my core work in theoretical reinforcement learning, but the connection is direct. It still centers on sequential decisions, model behavior over long horizons, and the relationship between predictive modeling and the quality of the resulting actions.

Public demo

The public demo makes the project easy to inspect without digging through a thesis or repository first. It is a useful showcase because visitors can immediately see whether the model behaves coherently and whether the "human-like" framing feels convincing in practice.

Open the embedded demo if you want to try it from this site, or launch it directly on Hugging Face.

What I would expand next

This note is a first project overview. The natural next additions would be a description of the training setup, the dataset construction, the evaluation criteria, and concrete game examples showing where the model looks convincingly human and where it breaks down.
